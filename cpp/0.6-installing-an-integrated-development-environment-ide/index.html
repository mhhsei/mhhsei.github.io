0.6 Installing an Integrated Development Environment (IDE)

Author: ALEX Date:JANUARY 14, 2023

source

集成開發環境 (IDE) 是一種軟件，包含開發、編譯、鏈接和調試程序所需的所有內容。

在典型的c++ IDE中，代碼編輯器可以進行行編號和語法高亮顯示。
許多(但不是全部)IDE包含c++編譯器和鏈接器，IDE知道如何與它們交互，以便將源代碼轉換為可執行文件。
當需要調試程序時，可以使用集成調試器。

此外，IDE 通常捆綁了許多其他有用的編輯功能，例如集成幫助、名稱完成、自動格式化，有時還有版本控制系統。
因此，雖然您可以單獨完成所有這些事情，但安裝 IDE 並從一個界面訪問它們要容易得多。

所以，讓我們安裝一個吧!顯而易見的下一個問題是，是安裝哪一個IDE?
許多ide都是免費的(在價格上)，如果您願意，可以安裝多個ide，因此這裡沒有錯誤的決定。我們將在下面推薦一些我們最喜歡的。

如果您有其他IDE，當然也可以使用他們。
我們在這些教程中展示的概念通常適用於任何像樣的現代IDE。
然而，不同的IDE使用不同的名稱、佈局、鍵映射等，所以你可能不得不在IDE中做一些搜索來找到等效的功能。

小貼士:為了獲得本教程的最大價值，我們建議安裝一個帶有c++ 17編譯器的IDE。

如果您被限制使用只支持c++ 14或c++ 11的編譯器(由於教育或業務限制)，大多數課程和示例仍然可以工作。然而，如果你遇到了使用c++ 17(或更新版本)概念的課程，而你使用的是較舊的語言編譯器，你將不得不跳過它或將其翻譯到你的版本，這可能很容易，也可能不容易。
你不應該使用任何不支持至少c++ 11(通常被認為是c++的現代最低規範)的編譯器。

我們建議安裝最新版本的編譯器。如果你不能使用最新的版本，以下是支持c++ 17的編譯器的絕對最低版本:

  • GCC/G++ 7
  • Clang++ 8
  • Visual Studio 2017 15.7

如果你在Windows 10機器上開發，那麼我們強烈建議下載Visual Studio 2022 Community。
運行安裝程序後，您最終會看到一個屏幕，詢問您希望安裝什麼樣的工作負載。
選擇使用c++進行桌面開發。
如果不這樣做，則c++功能將不可用。

在屏幕右側選擇默認選項應該沒問題，但請確保選擇了 Windows 10 SDK。

Visual Studio Workload

如果磁盤空間和/或下載大小是一個挑戰，那麼 Microsoft 仍然提供 Visual Studio Express 2017 for Windows Desktop，您可以在頁面底部找到它。

Code::Blocks (for Linux or Windows)

如果您在 Linux 上進行開發（或者您在 Windows 上進行開發但希望編寫可以輕鬆移植到 Linux 的程序），我們推薦使用 Code::Blocks。
Code::Blocks 是一個免費、開源、跨平台的 IDE，可以在 Linux 和 Windows 上運行。

For Windows users

確保獲得捆綁了 MinGW 的 Code::Blocks 版本（它應該是文件名以 mingw-setup.exe 結尾的版本）。
這將安裝 MinGW，其中包括 GCC C++ 編譯器的 Windows 端口：
code:block MinGW Windows 下載

Code::Blocks 20.03附帶了一個過時的MinGW版本，只支持c++ 17(目前是最新版c++的一個版本)。如果您想使用最新版本的c++ (c++ 20)，您將需要更新MinGW。要做到這一點，請遵循以下步驟:

  1. 按照上面的方法安裝 Code::Blocks。
  2. 關閉Code::Blocks 如果它是打開的。
  3. 打開 Windows 文件資源管理器（鍵盤快捷鍵 Win-E）。
  4. 導航到您的 Code::Blocks 安裝目錄（可能是 C:\Program Files (x86)\CodeBlocks）。
  5. 將“MinGW”目錄重命名為“MinGW.bak”（以防出錯）。
  6. 打開瀏覽器並導航至 https://winlibs.com/。
  7. 下載更新版本的 MinGW。您可能想要 Release Versions -> UCRT Runtime -> LATEST -> Win64 -> without LLVM/Clang/LLD/LLDB -> Zip archive 下的版本。
  8. 將“mingw64”文件夾解壓縮到您的 Code::blocks 安裝目錄。
  9. 將“mingw64”重命名為“MinGW”。

一旦確認更新後的編譯器可以工作，就可以刪除舊文件夾(MinGW.bak)。

For Linux users

某些 Linux 安裝可能缺少使用 Code::Blocks 運行或編譯程序所需的依賴項。

基於 Debian 的 Linux 用戶（例如使用 Mint 或 Ubuntu 的用戶）可能需要安裝 build-essential 軟件包。
為此，請從終端命令行鍵入：sudo apt-get install build-essential。

Arch Linux 用戶可能需要安裝 base-devel 包。

使用其他 Linux 變體的用戶將需要確定他們的等效包管理器和包是什麼。
當您第一次啟動 Code::Blocks 時，您可能會看到一個編譯器自動檢測對話框。
如果是這樣，請確保將 GNU GCC 編譯器設置為默認編譯器，然後選擇“確定”按鈕。

Compilers Auto Detection dialog

問:如果我得到一個“無法在您的GNU GCC編譯器的配置搜索路徑中找到可執行的編譯器”錯誤，我該怎麼辦

嘗試以下操作：

  1. 如果您使用的是 Windows，請確保您已經下載了 Code::Blocks WITH MinGW 版本。這是名字中帶有“mingw”的那個。
  2. 嘗試轉到設置、編譯器，然後選擇“重置為默認值”。
  3. 嘗試轉到設置、編譯器、工具鏈可執行文件選項卡，並確保“編譯器的安裝目錄”設置為 MinGW 目錄（例如 C:\Program Files (x86)\CodeBlocks\MinGW）。
  4. 嘗試完全卸載，然後重新安裝。
  5. 嘗試不同的編譯器。

Visual Studio Code (for experienced Linux, macOS, or Windows users)

Visual Studio Code（也稱為“VS Code”，不要與同名的“Visual Studio Community”混淆）是一種代碼編輯器，是經驗豐富的開發人員的熱門選擇，因為它快速、靈活、開源，適用於多種編程語言，並且可用於許多不同的平台。

缺點是 VS Code 比此列表中的其他選擇更難正確配置（在 Windows 上，也更難安裝）。
在繼續之前，我們建議您閱讀下面鏈接的安裝和配置文檔，以確保您理解並適應所涉及的步驟。

Other macOS IDEs

其他流行的 Mac 選擇包括 Xcode（如果您可以使用）和 Eclipse 代碼編輯器。默認情況下，Eclipse 未設置為使用 C++，您需要安裝可選的 C++ 組件。
雖然 Visual Studio for Mac 已經發布，但截至 2022 年 8 月，它不支持 C++，因此目前我們不推薦它。

Other compilers or platforms

問：我可以使用基於網絡的編譯器嗎？
是的，對於某些事情。在您的 IDE 下載期間（或者如果您不確定是否要安裝一個），您可以使用基於 Web 的編譯器繼續本教程。我們推薦以下之一：

  • TutorialsPoint
  • Wandbox（可選擇不同版本的GCC或Clang）
  • Godbolt（可以看到程序集）
    基於 Web 的編譯器非常適合涉獵和簡單的練習。但是，它們的功能通常非常有限——許多不允許您創建多個文件或有效地調試您的程序。如果可以，您將希望遷移到完整的 IDE。
    問：我可以使用命令行編譯器（例如 Linux 上的 g++）嗎？

是的，但我們不建議初學者使用它。您需要找到自己的編輯器並查找如何在其他地方使用它。使用命令行調試器不像集成調試器那麼容易，並且會使調試程序更加困難。

問：我可以使用其他代碼編輯器或 IDE，例如 Eclipse、Sublime 或 Notepad++ 嗎？

是的，但我們不建議初學者使用它。有許多很棒的代碼編輯器和 IDE，可以配置為支持多種語言，並允許您混合和匹配插件以自定義您喜歡的體驗。然而，許多這些編輯器和 IDE 需要額外的配置來編譯 C++ 程序，並且在此過程中可能會出現很多錯誤。對於初學者，我們推荐一些開箱即用的東西，這樣您就可以花更多的時間學習編碼，而花更少的時間去弄清楚為什麼您的代碼編輯器無法與您的編譯器或調試器一起正常工作。

要避免的 IDE

您應該完全避免使用以下 IDE，因為它們至少不支持 C++11，根本不支持 C++，或者不再積極支持或維護：

• Borland Turbo C++——不支持 C++11
• Visual Studio for Mac -- 不支持 C++

當存在支持現代 C++ 的輕量級免費替代方案時，沒有充分的理由使用過時或不受支持的編譯器。
當出現問題時（也就是當 IDE 代表“我什至……”時）
IDE 安裝似乎會導致相當多的問題。安裝可能會徹底失敗（或者安裝可能會工作，但由於配置問題，IDE 在您嘗試使用它時會出現問題）。如果遇到此類問題，請嘗試卸載 IDE（如果它首先安裝），重新啟動計算機，暫時禁用防病毒軟件或反惡意軟件，然後再次嘗試安裝。
如果此時您仍然遇到問題，您有兩種選擇。更簡單的選擇是嘗試不同的 IDE。另一種選擇是解決問題。不幸的是，安裝和配置錯誤的原因多種多樣，並且特定於 IDE 軟件本身，我們無法就如何解決此類問題提供有效建議。在這種情況下，我們建議將您遇到的錯誤消息或問題複製到您最喜歡的搜索引擎（例如 Google 或 Duck Duck Go）中，並嘗試從某個不可避免地遇到相同問題的可憐人那裡找到論壇帖子。通常會有關於您可以嘗試解決問題的建議。

接著往下說

安裝 IDE 後（如果事情沒有按預期進行，這可能是最困難的步驟之一），或者如果您暫時使用基於 Web 的編譯器，您就可以編寫您的第一個程序了！
