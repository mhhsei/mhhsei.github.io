0.7 — Compiling your first program

在我們編寫第一個程序之前，我們需要學習如何在我們的集成開發環境 (IDE) 中創建新程序。
在本課中，我們將介紹如何做到這一點，您還將編譯並執行您的第一個程序！

Projects

要在 IDE 中編寫 C++ 程序，我們通常首先創建一個新項目（稍後我們將向您展示如何執行此操作）。
項目是一個容器，其中包含所有源代碼文件、圖像、數據文件等……生成可運行或使用的可執行文件（或庫、網站等）所需的這些文件。
該項目還保存各種 IDE、編譯器和鏈接器設置，並記住您離開的地方，以便您稍後重新打開項目時，IDE 的狀態可以恢復到您離開的地方。
當您選擇編譯程序時，項目中的所有 .cpp 文件都將被編譯和鏈接。
每個項目對應一個程序。
當您準備好創建第二個程序時，您要么需要創建一個新項目，要么覆蓋現有項目中的代碼（如果您不想保留它）。
項目文件通常是特定於 IDE 的，因此為一個 IDE 創建的項目需要在不同的 IDE 中重新創建。

Best practice

為您編寫的每個新程序創建一個新項目。

Console projects

當您創建一個新項目時，系統通常會詢問您要創建什麼類型的項目。
我們將在本教程中創建的所有項目都是控制台項目。
控制台項目意味著我們將創建可以從 Windows、Linux 或 Mac 控制台運行的程序。

Windows console

默認情況下，控制台應用程序沒有圖形用戶界面 (GUI)，它們將文本打印到控制台，從鍵盤讀取輸入，並編譯成獨立的可執行文件。
這非常適合學習 C++，因為它可以將復雜性降到最低，並確保在各種系統上都能正常工作。

如果您以前從未使用過控制台，或者不知道如何訪問它，請不要擔心。
我們將通過我們的 IDE 編譯和啟動我們的程序（必要時將調用控制台）。

Workspaces / solutions

當您為您的程序創建一個新項目時，許多 IDE 會自動將您的項目添加到“工作區”或“解決方案”（術語因 IDE 而異）。工作區或解決方案是一個容器，可以容納一個或多個相關項目。例如，如果您正在編寫遊戲並希望為單人遊戲和多人遊戲分別創建可執行文件，則需要創建兩個項目。這兩個項目完全獨立是沒有意義的——畢竟，它們是同一個遊戲的一部分。最有可能的是，每個都將配置為單個工作區/解決方案中的單獨項目。
儘管您可以將多個項目添加到單個解決方案中，但我們通常建議為每個程序創建一個新的工作區或解決方案，尤其是在學習時。它更簡單，出錯的可能性更小。

問：什麼是預編譯頭，我們為什麼要關閉它們？

出於這個原因，我們建議一開始就關閉預編譯頭文件，只有在您發現編譯時間不佳時才啟用它們。

如果您的程序運行但控制台窗口閃爍並立即關閉
當控制台程序運行時，控制台窗口將打開，程序的任何輸出都將寫入控制台窗口。
當程序完成運行時，大多數現代 IDE 將保持控制台打開（直到您按下某個鍵），以便您可以在繼續之前檢查程序的結果。
但是，一些較舊的 IDE 會在程序運行結束時自動關閉控制台窗口。
這通常不是您想要的。
如果您的 IDE 自動關閉控制台窗口，可以使用以下兩個步驟來確保控制台在程序結束時暫停。

首先，添加或確保以下行位於程序頂部附近：

#include <iostream>
#include <limits>

基本上就放在其他 #include 的下方就可以了

其次，在 main() 函數的末尾添加以下代碼（就在 return 語句之前）：

std::cin.clear(); // reset any error flags
std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n'); // ignore any characters in the input buffer until we find a newline
std::cin.get(); // get one more char from the user (waits for user to press enter)

這將導致您的程序在繼續之前等待用戶按下 enter（您可能必須按下 enter 兩次），這將使您有時間在 IDE 關閉控制台窗口之前檢查程序的輸出。

其他解決方案，例如通常建議的系統（“暫停”）解決方案可能僅適用於某些操作系統，應避免使用。

如果控制台窗口根本沒有打開並且您的程序似乎沒有運行，則您的防病毒軟件或反惡意軟件也可能阻止了該程序的執行。
如果是這種情況，請嘗試暫時禁用掃描儀並查看問題是否解決。

我的 IDE 中的編譯、構建、重建、清理和運行/啟動選項之間有什麼區別？
在第 0.5 課 -- 編譯器、鏈接器和庫簡介中，我們展示了要生成可以運行的可執行文件，程序中的每個代碼文件都被編譯成目標文件，然後將目標文件鏈接到可執行文件中.

編譯代碼文件時，您的 IDE 可能會緩存生成的目標文件。
這樣，如果以後再次編譯程序，任何沒有修改的代碼文件都不需要重新編譯——上次緩存的目標文件可以重新使用。
這可以顯著加快編譯時間（以一點磁盤空間為代價）。

考慮到這一點，以下是每個選項通常的作用：

  • Build 編譯項目或工作區/解決方案中所有修改過的代碼文件，然後將目標文件鏈接到可執行文件中。
    如果自上次構建以來沒有修改任何代碼文件，則此選項不執行任何操作。
  • Clean 刪除所有緩存的對象和可執行文件，因此在下次構建項目時，將重新編譯所有文件並生成新的可執行文件。
  • Rebuild 執行“clean”，然後執行“build”。
  • Compile 重新編譯單個代碼文件（不管它之前是否已被緩存）。此選項不調用鏈接器或生成可執行文件。
  • Run/start 執行先前構建的可執行文件。某些 IDE（例如 Visual Studio）會在執行“運行”之前調用“構建”，以確保您運行的是最新版本的代碼。否則（例如 Code::Blocks）將只執行先前的可執行文件。

雖然我們非正式地談論“編譯”我們的程序，但要實際編譯我們的程序，我們通常會在我們的 IDE 中選擇“構建”（或“運行”）選項來這樣做。
