Introduction to C/C++

source

Author ALEX DateJULY 7, 2021

在c++之前，有C語言。C語言是1972年由Dennis Ritchie在貝爾電話實驗室開發的，主要作為一種系統編程語言(一種用來編寫操作系統的語言)。
Ritchie 的主要目標是開發一種易於編譯、允許高效訪問內存、生成高效代碼並且自包含（不依賴於其他程序）的簡約語言。
作為一種高級語言，它被設計為給予程序員大量的控制，同時仍然鼓勵平臺(硬件和操作系統)獨立性(也就是說，代碼不必為每個平臺重寫)。
C 最終變得如此高效和靈活，以至於在 1973 年，Ritchie 和 Ken Thompson 使用 C 重寫了大部分 Unix 操作系統。
許多以前的操作系統都是用彙編語言編寫的。

與生成只能在特定 CPU 上運行的程序的彙編不同，C 具有出色的可移植性，使 Unix 可以輕鬆地在許多不同類型的計算機上重新編譯並加速其採用。
C 和 Unix 命運連在一起，C 的流行部分與 Unix 作為操作系統的成功有關。
1978 年，Brian Kernighan 和 Dennis Ritchie 出版了一本名為《The C Programming Language》的書。
這本書通常被稱為 K&R（以作者的姓氏命名），為該語言提供了一個非正式的規範，並成為事實上的標準。
當需要最大限度的可移植性時，程序員會堅持 K&R 中的建議，因為當時大多數編譯器都是按照 K&R 標準實現的。

1983年，美國國家標準協會(ANSI)成立了一個委員會來建立C語言的正式標準。
1989 年，他們完成並發布了 C89 標準，通常稱為 ANSI C。
1990年，國際標準化組織(ISO)採用了ANSI C(且有一些小的修改)。
這個版本的 C 被稱為 C90。

編譯器最終成為 ANSI C/C90 兼容的，並且需要最大可移植性的程序被編碼為該標準。

1999 年，ANSI 委員會發布了一個新版本的 C，稱為 C99。
C99 採用了許多功能，這些功能已經作為擴展進入編譯器，或者已經在 C++ 中實現。

C++

C++（發音為 see plus plus）由貝爾實驗室的 Bjarne Stroustrup 作為 C 的擴展於 1979 年開發。
C++ 為 C 語言添加了許多新功能，也許最好將其視為 C 的超集，儘管嚴格來說這並非如此（因為 C99 引入了一些 C++ 中不存在的功能）。
C++ 聲名鵲起主要是因為它是一種面向對象的語言。
至於什麼是“對象”以及它與傳統編程方法有何不同，我們將在後面的章節中介紹。

C++ 於 1998 年由 ISO 委員會標準化（這意味著 ISO 標準委員會批准了一份描述 C++ 語言的文檔，以幫助確保所有編譯器都遵守同一組標準）。
該語言的一個小更新於 2003 年發布（稱為 C++03）。

從那時起，c++語言進行了四個主要的更新(c++ 11、c++ 14、c++ 17和c++ 20)，每一次更新都增加了額外的功能。c++ 11特別增加了大量的新功能，並被廣泛認為是該語言的新基準版本。隨著c++ 23的特性集已經開始制定，預計每隔三年左右就會對該語言進行升級。

語言的每一個新的正式版本都被稱為語言標準(或語言規範)。標準以它們發佈的年份命名。例如，沒有c++ 15，因為2015年沒有新的標準。

C和C++的哲學

C 和 C++ 的底層設計哲學可以概括為“相信程序員”——這既美妙又危險。
C++旨在讓程序員有高度的自由去做他們想做的事。
然而，這也意味著該語言通常不會阻止你做一些沒有意義的事情，因為它會假設你這樣做是出於某種它不理解的原因。如果不注意，新程序員可能會掉入很多陷阱。這是為什麼知道在 C/C++ 中不應該做什麼幾乎與知道應該做什麼一樣重要的主要原因之一。

問:c++擅長什麼

C++ 在需要高性能和精確控制內存和其他資源的情況下表現出色。以下是最有可能用 C++ 編寫的幾種常見應用程序類型：

  • 視頻遊戲
  • 實時系統（例如用於運輸、製造等……）
  • 高性能金融應用程序（例如高頻交易）
  • 圖形應用和模擬
  • 生產力/辦公應用
  • 嵌入式軟件
  • 音視頻處理
  • 人工智能和神經網絡
